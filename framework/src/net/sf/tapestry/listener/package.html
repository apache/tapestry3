Support classes that allows an object to expose listener methods instead of listener properties.

Normally, a listener property must be an object that implements either {@link net.sf.tapestry.IActionListener} or {@link net.sf.tapestry.IDirectListener}. This can be cumbersome, in practice, as it typically involves creating an anonymous inner class.

Using this mechanism, classes can instead implement listener methods. A listener method takes one of two forms:

  • public void method-name({@link net.sf.tapestry.IRequestCycle} cycle)
    throws {@link net.sf.tapestry.RequestCycleException}
  • public void method-name(String[] context, {@link net.sf.tapestry.IRequestCycle} cycle)
    throws {@link net.sf.tapestry.RequestCycleException}

The first option can act like either an {@link net.sf.tapestry.IActionListener} or a {@link net.sf.tapestry.IDirectListener}, the second must be a {@link net.sf.tapestry.IDirectListener}. In both cases, the throws clause is optional.

Tapestry will create an appropriate listener object that will invoke the corresponding method.
