Components and classes that provide specialized, validating text fields.
  • Add new constraints:
    • Fields may be required or optional
    • For {@link ValidatingTextField}, a minimum length (after whitespace is stripped)
    • For {@link IntegerField}, a minimum and maximum value
    • For {@link DateField}, a minimum and maximum date
  • Fields that are in error can be marked with a icon or label.
  • When rendering, the first field that is either required and null, or in error is identified. This identified field is focused and selected (effectively, the cursor jumps right to the field). @author Howard Ship hship@primix.com
