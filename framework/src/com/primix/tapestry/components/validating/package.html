Components and classes that provide specialized, validating text fields. The interface {@link com.primix.tapestry.components.validating.IValidatingTextField} provides the general outline.
  • Add new constraints:
    • Fields may be required or optional
    • For {@link com.primix.tapestry.components.validating.ValidatingTextField}, a minimum length (after whitespace is stripped)
    • For {@link com.primix.tapestry.components.validating.IntegerField}, a minimum and maximum value
    • For {@link com.primix.tapestry.components.validating.DateField}, a minimum and maximum date
  • Fields that are in error can be marked with an icon or label
  • When rendering, the first field that is either required and null, or in error is identified. This identified field is focused and selected (effectively, the cursor jumps right to the field).

Since presenting errors to the user is completely specific to the application, a second interface, {@link com.primix.tapestry.components.validating.IValidationDelegate} provides the bridge between these components and the application they are being used in. @author Howard Ship hship@primix.com
